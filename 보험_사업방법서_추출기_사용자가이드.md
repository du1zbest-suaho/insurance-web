## 1. 프로그램 소개

**보험 사업방법서 추출기**는 PDF 형식의 보험 사업방법서에서 아래 4가지 테이블 데이터를 자동으로 추출하여 xlsx 파일로 변환해 주는 도구입니다.

| 테이블 코드 | 이름 | 내용 |
|------------|------|------|
| S00026 | 가입가능나이 | 피보험자 최저/최고 가입 나이 |
| S00027 | 보기납기 | 보험기간·납입기간 조합 |
| S00028 | 납입주기 | 월납/분기납/반기납/연납/일시납 |
| S00022 | 보기개시나이 | 연금/보험 개시 나이 |

### 주요 기능

- PDF 텍스트 자동 추출 및 규칙 기반 파싱
- 여러 PDF 파일 동시 처리 (폴더 단위 업로드 지원)
- 실시간 진행 상황 표시
- 기존 GT(Ground Truth) 데이터와 자동 비교 → PASS/FAIL 판정
- 결과 xlsx 개별/전체 ZIP 다운로드

## 2. 시작하기 — 프로그램 실행

### 방법 A: exe 파일로 실행 (배포본)

1. 배포받은 `insurance-extractor` 폴더 전체를 원하는 위치에 복사합니다.
2. 폴더 안의 **`insurance-extractor.exe`** 를 더블클릭합니다.
3. 잠시 후 **Microsoft Edge 브라우저가 자동으로 열리며** 화면이 표시됩니다.

> **주의:** `insurance-extractor.exe` 파일 하나만 복사하면 실행되지 않습니다.
> 반드시 **폴더 전체**를 복사하세요.

```
insurance-extractor/          ← 이 폴더 전체를 복사
├── insurance-extractor.exe   ← 실행 파일
└── _internal/                ← 필수 파일 (건드리지 마세요)
```

### 방법 B: Python 개발 환경에서 실행

```bash
cd insurance-web
python run.py
```

브라우저가 자동으로 열리지 않으면 직접 `http://127.0.0.1:8765` 주소를 입력하세요.

## 4. 단계별 사용 방법

### STEP 1. 매핑 파일 업로드 *(필수)*

매핑 파일은 **PDF 파일명과 상품 코드(ITCD/DTCD)를 연결**하는 핵심 파일입니다.

**파일명 형식:** `판매중_상품구성_사업방법서_매핑.xlsx`

**업로드 방법:**
- 매핑 파일 영역의 점선 박스를 **클릭** → 파일 선택
- 또는 파일을 점선 박스 위로 **드래그 앤 드롭**

업로드 성공 시 우측 상단 뱃지가 `매핑 미로드`(빨강) → **`매핑 로드됨`(초록)** 으로 바뀝니다.

> 매핑 파일 없이는 추출을 시작할 수 없습니다.

---

### STEP 2. GT 파일 업로드 *(선택)*

GT(Ground Truth) 파일은 **기존에 올바르게 등록된 기준 데이터**입니다.
업로드하면 추출 결과와 자동 비교하여 **PASS/FAIL을 판정**합니다.

**파일명 형식 (아래 중 해당하는 파일 업로드):**

| 파일명 (예시) | 비교 테이블 |
|--------------|------------|
| `판매중_가입나이정보.xlsx` 또는 `판매중_가입가능나이.xlsx` | S00026 |
| `판매중_보기납기정보.xlsx` | S00027 |
| `판매중_납입주기정보.xlsx` | S00028 |
| `판매중_보기개시나이정보.xlsx` | S00022 |

**업로드 방법:**
- GT 파일 영역 클릭 → 여러 파일 동시 선택 가능
- 업로드 성공 시 `S00026 GT`, `S00027 GT` 등 뱃지가 상단에 표시됩니다.

> GT 파일이 없으면 비교 없이 추출만 진행됩니다.

---

### STEP 3. 템플릿 파일 업로드 *(선택)*

출력할 xlsx의 **양식(서식)을 지정**하는 파일입니다.

**파일명 형식:** 파일명에 `S00026`, `S00027`, `S00028`, `S00022` 중 하나가 포함되어야 합니다.
예) `S00026_업로드양식.xlsx`, `가입나이_S00027_template.xlsx`

> 템플릿이 없으면 기본 양식으로 출력됩니다.

---

### STEP 4. PDF 파일 업로드 *(필수)*

처리할 보험 사업방법서 PDF를 업로드합니다.

**방법 1 — 파일 선택:**
- PDF 영역 클릭 → 여러 파일 동시 선택 가능 (Ctrl+클릭)

**방법 2 — 폴더 전체 선택 (Edge/Chrome 전용):**
- `📁 폴더 전체 선택` 버튼 클릭 → 폴더 선택
- 폴더 안의 모든 PDF가 자동으로 업로드됩니다.

**방법 3 — 드래그 앤 드롭:**
- 파일 탐색기에서 PDF를 화면으로 끌어다 놓으면 됩니다.

업로드된 파일은 **칩(태그)** 형태로 표시됩니다. `×` 버튼으로 개별 삭제 가능합니다.

---

### STEP 5. 추출 테이블 선택

처리할 테이블 종류를 선택합니다. (기본값: 전체 체크)

| 체크박스 | 테이블 | 처리 내용 |
|---------|-------|----------|
| S00026 가입가능나이 | 가입나이 테이블 추출 | 성별·최저·최고나이 |
| S00027 보기납기 | 보험기간·납입기간 추출 | 납기 조합 |
| S00028 납입주기 | 납입 방법 추출 | 월납/연납 등 |
| S00022 보기개시나이 | 개시나이 추출 | 연금 개시 나이 |

필요한 항목만 체크 해제하면 해당 테이블은 건너뜁니다.

---

### STEP 6. 추출 시작

`▶ 추출 시작` 버튼을 클릭합니다.

> 버튼이 비활성화(회색)되어 있으면 **매핑 파일** 또는 **PDF 파일**이 누락된 상태입니다.

## 5. 처리 결과 확인

### 실시간 진행 상태

처리 중 각 파일별로 단계 뱃지가 순서대로 활성화됩니다:

```
1.추출 → 2.매핑 → 3.룰 → 4.변환 → 5.GT비교 → 7.xlsx
```

| 뱃지 색상 | 의미 |
|---------|------|
| 회색 | 대기 중 |
| 파란색(깜빡) | 현재 처리 중 |
| 초록색 | 완료 |
| 빨간색 | 오류 발생 |

### 최종 상태

| 상태 | 의미 | 색상 |
|------|------|------|
| ✅ PASS | GT 비교 결과 모든 행 일치 (누락 없음) | 초록 행 |
| ❌ FAIL | GT 비교 결과 누락 행 존재 | 빨간 행 |
| 신규 | GT 파일 없이 처리 완료 | 파란 행 |
| 오류 | 처리 중 오류 발생 | — |

### 상단 통계 카드

| 카드 | 설명 |
|------|------|
| 총 파일 | 업로드된 전체 PDF 수 |
| PASS | GT 비교 통과 파일 수 |
| FAIL | GT 비교 실패 파일 수 |
| 신규 | GT 없이 완료된 파일 수 |
| 오류 | 처리 실패 파일 수 |
| 경과시간 | 처리 시작부터 경과 시간 |

---

### 미리보기 패널

결과 테이블에서 **파일 행을 클릭**하면 하단에 미리보기 패널이 열립니다.

**왼쪽: 추출 근거 텍스트**
- PDF에서 추출된 원문 텍스트
- 핵심 키워드가 **노란색으로 하이라이트** 표시

**오른쪽: 추출 결과 테이블**
- 추출·변환된 데이터 행 목록

GT 파일이 있는 경우 행 색상으로 비교 결과를 표시합니다:

| 행 색상 | 의미 |
|--------|------|
| 초록색 배경 | GT와 일치하는 행 |
| 파란색 배경 | 새로 추출된 행 (GT에 없음) |
| 노란색 배경 *기울임* | GT에는 있으나 추출되지 않은 행 (누락) |

## 6. 결과 다운로드

### 개별 파일 다운로드

결과 테이블 우측 **`S26`, `S27`, `S28`, `S22`** 버튼 클릭 → 해당 테이블 xlsx 다운로드

또는 미리보기 패널 우측 상단 **`xlsx ⬇`** 버튼 클릭

### 전체 ZIP 다운로드

처리 완료 후 나타나는 **`⬇ 전체 ZIP 다운로드`** 버튼 클릭
→ 모든 처리 결과 xlsx가 하나의 zip 파일로 다운로드됩니다.

## 7. GT 비교 기능 (PASS/FAIL 판정)

### GT 비교란?

현재 시스템에 등록된 **기준 데이터(GT)**와 이번에 추출한 결과를 행 단위로 비교하여,
데이터가 올바르게 추출됐는지 자동으로 검증하는 기능입니다.

### 비교 기준 필드

| 테이블 | 비교 기준 필드 |
|--------|--------------|
| S00026 | 보험기간코드, 납입기간코드, 성별코드, 최저나이, 최고나이 |
| S00027 | 보험기간코드, 납입기간코드 |
| S00028 | 납입주기코드 |
| S00022 | 최저나이, 최고나이 |

### PASS/FAIL 판정 기준

- **PASS**: GT에 있는 모든 행이 추출 결과에 존재 (누락 0건)
- **FAIL**: GT에 있는 행 중 1건 이상 추출 결과에 없음

### GT 요약 정보 읽기

미리보기 패널 상단에 표시되는 요약 예시:

```
❌ FAIL    15일치  3신규  2누락
```

| 항목 | 의미 |
|------|------|
| 일치 | GT와 완전히 같은 행 수 |
| 신규 | 이번에 새로 추출된 행 (GT에 없음) |
| 누락 | GT에 있어야 하는데 추출되지 않은 행 |

## 8. 자주 묻는 질문 (FAQ)

**Q. 프로그램 실행 시 브라우저가 열리지 않아요.**
A. Edge 브라우저가 설치되어 있는지 확인하세요. Edge가 없으면 Chrome 등 다른 브라우저에서 직접 `http://127.0.0.1:8765` 를 입력하세요.

**Q. `▶ 추출 시작` 버튼이 비활성화되어 있어요.**
A. 상단에 `매핑 미로드` 뱃지가 빨간색이면 매핑 파일을 먼저 업로드해야 합니다. PDF 파일도 업로드되어 있는지 확인하세요.

**Q. GT 파일을 올렸는데 비교가 안 돼요.**
A. GT 파일명에 `가입나이`, `보기납기`, `납입주기`, `보기개시나이` 중 하나가 포함되어야 합니다. 파일명을 확인하세요.

**Q. 같은 PDF를 여러 번 업로드했어요.**
A. 중복 파일은 자동으로 무시됩니다. 잘못 올린 파일은 칩의 `×` 버튼으로 삭제하세요.

**Q. 처리 중에 프로그램을 닫았어요.**
A. 재실행 후 파일을 다시 업로드하고 처음부터 진행해야 합니다.
(처리 중 작업은 메모리에만 저장되므로 재시작 시 초기화됩니다.)

**Q. xlsx 결과 파일이 다운로드되지 않아요.**
A. 해당 PDF 처리 상태가 `오류`인 경우 xlsx가 생성되지 않습니다. 오류 내용을 확인하고 PDF 파일이 정상인지 점검하세요.

**Q. 폴더 선택 버튼이 동작하지 않아요.**
A. 폴더 선택 기능은 **Edge** 또는 **Chrome** 브라우저에서만 동작합니다. Internet Explorer는 지원하지 않습니다.

**Q. 추출 규칙을 수정했는데 반영이 안 돼요.**
A. `sync_rules.py`를 실행하여 규칙 파일을 동기화하세요.
exe 배포본의 경우 규칙 파일(`_internal/app/rules/extraction_rules.py`)을 직접 수정하면 재시작 없이 자동으로 반영됩니다.

```bash
# 개발 환경에서 규칙 동기화
python sync_rules.py
```

## 9. 오류 해결 방법

### 오류: 프로그램이 실행되지 않음

- `insurance-extractor.exe`만 복사하지 않았는지 확인 → **폴더 전체 복사** 필요
- Windows 보안 경고 발생 시: `추가 정보` → `실행` 클릭

### 오류: 브라우저에서 `사이트에 연결할 수 없음`

- 프로그램이 완전히 시작될 때까지 10초 정도 기다린 후 다시 시도
- 이미 같은 포트로 다른 프로그램이 실행 중일 수 있음
  → 기존 프로세스 종료 후 재실행

### 오류: 파일 처리 중 `오류` 상태

가능한 원인:
1. **PDF가 스캔본(이미지 PDF)인 경우** — 텍스트 추출 불가. 텍스트 레이어가 있는 PDF 필요
2. **PDF가 암호화된 경우** — 암호 해제 후 재시도
3. **매핑 파일에 해당 상품이 없는 경우** — 매핑 파일에 상품 코드 추가 필요

### 오류: GT 비교 결과가 이상함

- GT 파일의 DTCD 코드와 매핑 파일의 코드가 일치하는지 확인
- GT 파일 형식이 올바른지(컬럼명 확인) 점검

## 부록: 파일 준비 체크리스트
